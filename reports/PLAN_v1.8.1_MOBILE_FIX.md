# 🔧 v1.8.1 移动端布局修复计划

**创建日期**: 2026-03-11  
**问题报告**: 用户从手机打开网页无法进入主界面  
**优先级**: P0（阻塞性问题）

---

## 🐛 问题描述

### 用户反馈
- 从手机浏览器访问 https://HongyiLU.github.io/cyber-tetris-2077
- 无法看到主界面（开始按钮等）
- 可能原因：移动端布局问题导致主菜单内容被遮挡或挤到屏幕外

### 问题分析
**根本原因**: `ResponsiveLayout.tsx` 组件的移动端布局顺序问题

**当前布局顺序**（错误）:
```
1. 游戏画布（scale 0.85）
2. 游戏信息
3. 虚拟按键
4. children（主菜单内容）← 被挤到最下面
```

**问题点**:
1. 主菜单内容（开始按钮等）放在虚拟按键后面
2. 游戏画布缩放 0.85 倍，在小屏幕上可能显示异常
3. 布局间距过大，浪费屏幕空间
4. 没有 `overflowX: hidden`，可能导致横向滚动

---

## ✅ 修复方案

### 方案 A：调整布局顺序（推荐）

**新布局顺序**:
```
1. children（主菜单内容）← 放在最前面
2. 游戏画布（不缩放，自适应宽度）
3. 游戏信息
4. 虚拟按键 ← 放在最底部
```

**修改文件**: `src/components/ui/ResponsiveLayout.tsx`

**具体改动**:
1. 将 `children` 移到布局顶部
2. 移除游戏画布的 `scale(0.85)` 变换
3. 设置游戏画布最大宽度 320px（自适应）
4. 添加 `overflowX: hidden` 防止横向滚动
5. 减小布局间距（15px → 10px）

### 方案 B：添加移动端专用菜单（备选）

如果方案 A 无法解决问题，考虑：
- 为移动端创建专用的简化菜单
- 使用绝对定位确保菜单可见
- 添加"开始游戏"浮动按钮
