## 📋 通用开发流程（5 步）

这是我们确定的标准开发流程，所有版本开发都必须遵循：

```
1. 需求分析
   ↓
2. 功能实现
   ↓
3. 代码审查
   ↓
4. 功能测试
   ↓
5. 整合发布
```

### 1️⃣ 需求分析

**工作内容**:
- 分析功能需求
- 确定功能范围和边界
- 制定验收标准
- 创建计划文档（如 `PLAN_vX.X.X.md`）

**产出物**:
- 📄 需求/计划文档

---

### 2️⃣ 功能实现

**工作内容**:
- 编写代码实现功能
- 创建 UI 组件
- 编写单元测试
- TypeScript 类型定义

**产出物**:
- 💻 源代码
- 🧪 单元测试
- 📝 代码注释

---

### 3️⃣ 代码审查

**工作内容**:
- 审查代码质量
- 检查类型安全
- 评估测试覆盖率
- 提出改进建议
- 给出审查评分

**产出物**:
- 📄 代码审查报告
- ⭐ 审查评分
- 📋 改进建议列表

---

### 4️⃣ 功能测试

**工作内容**:
- 运行单元测试 (`npm test`)
- TypeScript 编译检查 (`npx tsc --noEmit`)
- 生产构建测试 (`npm run build`)
- 回归测试
- 性能测试（可选）

**产出物**:
- 📊 测试报告
- ✅ 测试通过确认

---

### 5️⃣ 整合发布

**工作内容**:
- 提交代码到 git (`git add`, `git commit`)
- 推送到远程仓库 (`git push`)
- 创建版本标签 (`git tag`, `git push --tags`)
- GitHub Pages 部署
- 编写发布说明和文档

**产出物**:
- 📦 Git 提交
- 🏷️ 版本标签
- 📄 发布说明
- 📊 测试报告

## 🎯 重要决策

1. **流程优先** - 必须遵循 5 步开发流程
2. **测试必须** - 新功能必须有单元测试
3. **审查必须** - 代码必须经过 Code Review
4. **文档必须** - 功能必须有实现报告和测试报告
