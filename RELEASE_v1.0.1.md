## 📋 问题描述

### 原始 Bug
硬降 (hardDrop) 功能存在严重问题：
- ❌ 方块无法下落，直接"卡"在半空中
- ❌ 方块没有被正确锁定到棋盘上
- ❌ 硬降后游戏状态异常

### 根本原因
原实现在下落循环**之前**就设置了 `pieceLocked = true`，导致：
1. `movePiece(0, 1)` 立即返回 `false`（因为检查到已锁定状态）
2. 方块无法下落
3. 没有调用 `lockPiece()` 进行真正锁定

---

## ✅ 修复内容

### 修改文件
`tetris-project/src/engine/GameEngine.ts`

### 修复代码
```typescript
/**
 * 硬降 - 方块瞬间下落到最低点后立即锁死
 * 
 * 流程：
 * 1. 循环下落直到无法继续移动（到达最低点）
 * 2. 调用 lockPiece() 将方块锁定到棋盘上（与自然落地流程完全一致）
 * 
 * @returns 下落的距离（行数）
 */
public hardDrop(): number {
  if (!this.currentPiece || this.gameOver || this.paused) return 0;

  let dropDistance = 0;
  
  // 循环下落直到无法继续移动（到达最低点）
  while (this.movePiece(0, 1)) {
    dropDistance++;
  }
  
  // 硬降后立即锁定方块（与自然落地流程完全一致）
  this.lockPiece();
  
  return dropDistance;
}
```

### 修复说明
1. ✅ 移除了错误的 `pieceLocked = true` 前置设置
2. ✅ 使用 `while` 循环让方块自然下落到底
3. ✅ 调用 `lockPiece()` 正确锁定方块
4. ✅ 返回下落距离用于分数计算
5. ✅ 添加了详细的 JSDoc 注释
